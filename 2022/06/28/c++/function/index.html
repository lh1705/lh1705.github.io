函数参数

默认参数

  1. 默认参数的优先级低于实参的优先级（默认参数位置传入实参时，默认参数被覆盖）
  2. 形参列表中，从默认参数位置起，其后的所有参数都必须为默认参数。
  3. 函数的声明与函数定义，其中只能有一个拥有默认参数（建议在声明中写）

占位参数

  1. 占位参数即只指定函数形参的数据类型，不指定形参名
  2. 占位参数也可以使用默认参数

参数传递

形参的类型决定了形参和实参的交互方式。如果形参是引用类型，它将绑定到对应的实 参上，其对应实参被引用传递；否则，将实参的值拷贝后赋给形参，形参和实参是 两个相互独立的对象，其对应实参被值传递。

值传递

值传递，不能直接传递实参对象，即只是向函数传递一个实参对象的副本，其在函数内 部对该对象的操作，其实是对该实参对象副本的操作，对于实参对象本身没有任何影响。

值传递虽然不能直接传递实参对象，但可以借助指针类型的形参传递实参对象的地 址。通过地址，来访问外部实参对象。

引用传递

引用传递，借助引用形参绑定到要传递给函数的外部对象上，此时通过对象别名直接访问外 部对象。

const 形参和实参

实参初始化形参时，忽略顶层 const （常量指针）。

形参中顶层 const 限定指针是常量（存的地址不能改变），但用指针作为形参，不就是 要向形参传入外部对象地址吗。此时用 const 限定不能改变指针的值（存的地址），那 还传入啥，所以此时忽略 const，完成形参初始化。

函数重载

重载条件

  1. 函数名相同
  2. 函数在同一个作用域下
  3. 函数形参列表的形参数据类型不同 / 个数不同 / 顺序不同

NOTE 函数的返回值不能作为函数重载的条件。

顶层 const 不可重载

实参初始化形式形参时忽略顶层 const，即在匹配函数时忽略顶层 const 进行匹配，进 而导致带顶层 const 和无 const 的函数是同一函数，造成函数二义性调用，导致不能 重载。

NOTE const int a 与 int const a 是等价的，都是限定指针指向的对象是常量。 非常量引用也是顶层 const，不能进行重载。

常量引用、底层 const 作为重载条件

非常量引用就是顶层 const，即限定指针是一个常量，同样不能作为重载条件。底层 const 限定指针指向的对象是一个常量，const 不忽略，可以作为重载条件。但常量引 用可以作为重载条件。
