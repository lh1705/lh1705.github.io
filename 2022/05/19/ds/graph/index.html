  • 简单路径。在路径序列中，顶点不重复出现的路径。

4 图的遍历

4.1 BFS

思想

  • 首先访问起始顶点 ，接着依次访问 的各个未被访问过的邻接点
  • 然后依次访问 的所有未被访问的邻接顶点 （之前的顶点 a 已经
    被访问    ）
  • 再从这些访问过的顶点出发，依次访问它们所有未被访问过的邻接顶点，直至图中所有顶点都被访问过为止。
  • 若此时图中尚有顶点未被访问 ，则另选图中一个未被访问的顶点作为始点，
    重复上述过程，直至图中所有顶点都被访问到为止。

实现

性能分析

  • 空间复杂度 (辅助队列，每个顶点都要进队）
  • 时间复杂度 邻接表 ，邻接矩阵

4.2 DFS

思想

  • 首先以某个顶点为起始点 v0，从该顶点 v0 出发，访问与其相邻的且未被访问的一个
    顶点     v1
  • 再访问与 v1 相邻的且未被访问的一个顶点 v2…
  • 重复上述过程，直到不能继续向下访问时，依次退回最近访问的结点（栈）
  • 若该顶点还有邻接点未被访问过，则从该点开始继续上述过程，直至访问完图的一个
    连通分量。
  • 若此时还有顶点未被访问，则从未被访问的某个顶点开始，重复上述步骤。(访问其他连
    通分量）

实现

性能分析

  • 空间复杂度 (递归工作栈）
  • 时间复杂度 邻接表 ，邻接矩阵

5 图的应用

5.1 最小生成树

  • 最小生成树，
  • 最小生成树的代价唯一
  • 最小生成树为无环图
  • 最小生成树是极小连通子图
  • 无向连通图必有最小生成树
  • 无向连通图可能有一个或多个最小生成树。(因为有可能存在多条权值相等的边）
  • 无向连通图没有权值相同的边，则其有唯一最小生成树
  • 无向连通图中有多条权值相等的边，其最小生成树不一定唯一（假设有两条权值相等
    的边，如果一条边会导致环出现，则这条边不能生成最小生成树）
  • 用 Prim 算法和 Kruskal 算法构造的最小生成树，可能相同，也可能不同

Prim 算法（无向网）

  • 基于贪心策略
  • 时间复杂度
  • 适用于边稠密的图 (不依赖于 )

Kruskal 算法（无向网）

  • 基于贪心策略
  • 时间复杂度
  • 适用于边稀疏而顶点较多的图（不依赖于 )

5.2 最短路径

最短路径路径问题：如果从有向图中某一顶点 (称为源点) 到达另一顶点 (称为终点)
的路径可能不止一条，如何找到一条路径使得沿此路径上各边上的权值总和达到最小。

带权有向图的最短路径问题一般分为两类：

  1. 单源最短路径。即求图中某一顶点到其他各顶点的最短路径。
  2. 求每对顶点间的的最短路径。
  • 最短路径可以是有向带环图

Dijkstra 算法 （单源最短路径）

  • Dijkstra 算法是基于贪心策略的
  • 边上带有负权值时，Dijkstra 不适用

Floyd 算法（每对顶点最短路径）

  • 基于贪心策略
  • 时间复杂度
  • 计算任意两点间的最短路径

AOV 网

用有向无环图（）表示一个工程，其顶点表示活动，用有向边 表示
活动 必须先于活动 进行这样一种关系，则将这种有向图称为顶点表示活
动的网络，记为 网。

拓扑排序

  • 有向无环图
  • 每个顶点出现且只出现一次
  • 若顶点 A 在序列排在顶点 B 的前面，则在图中不存在从顶点 B 到顶点 A 的路径
  • 每个 AOV 网都有一个或多个拓扑排序序列
  • 若一个顶点有多个直接后继（出度 > 1)，则拓扑序列不唯一
  • 对于一般图，若其邻接矩阵是三角矩阵，则存在拓扑序列(可能不唯一)
  • 时间复杂度。邻接表 ，邻接矩阵

5.5 关键路径

  • 增加任一关键活动的时间都会延长工程的工期
  • 缩短一条关键活动的时间不一定会缩短工程的工期
  • 仅有一条关键路径时，减少关键活动的时间会缩短工程的工期
  • 要缩短工程的工期，缩短时间的关键活动可以为非共同关键活动，
    (例如 a-b-c a-c-e b-d-f，缩短 cf 就可以，只要保证每一条都缩短就行）
  • 改变网上某一关键路径上的任一关键活动后，有可能产生不同的关键路径
    因为，有可能改变的是公共关键活动，关键路径不变

AOE 网

在带权有向图中，以顶点表示时间，以有向边表示活动，以边上的权值表示完成该活
动的开销（如完成活动所需的时间），其称之为用边表示活动的网络，简称 AOE 网。

关键路径

从源点到汇点的所有路径中，具有最大路径长度的路径，称之为关键路径。

  • 源点。在 AOE 网中仅有一个的入度为 0 的顶点。
  • 汇点。在 AOE 网中仅有一个的出度为 0 的顶点。
  • 关键活动。关键路径上的活动（有向边）
