数组

  • 数组的大小在编译阶段确定。(编译阶段为数组分配内存，必须明确大小)
  • 数组的维度必须是常量表达式。（constexpr）
  • 不允许使用一个数组初始化另一个数组。
  • 不能把一个数组直接赋值给另一个数组。
  • 数组初始化后，不能整体赋值。

3 内存分区模型

  • 代码区：存放函数体的二进制代码，由操作系统管理

  • 代码区：存放函数体的二进制代码，由操作系统管理

  • 全局区: 存放全局变量、静态变量以及常量，程序结束后由操作系统释放

  • 栈区：存放函数的参数值，局部变量等，由编译器自动分配释放

  • 堆区：由程序员自行分配和释放，若程序员不释放，程序结束后由操作系统回收

3.1 代码区

  1. 存放 CPU 相关的机械指令，即二进制代码
  2. 代码区是共享的，多次执行同一个程序，在内存中有一份代码
  3. 代码区是只读的，防止程序意外修改其指令

3.2 全局区

  1. 存放全局变量、静态变量以及常量区内容
  2. 常量区包含字符串常量和其他常量（const 全局常量）

Note 其他常量包括 const 修饰的全局变量以及 const 修饰的局部变量，其中 全局常量位于全局区，而局部常量位于栈区

3.3 栈区

存放函数内部的局部变量，以及函数的形参

Note 不要返回局部变量的地址，栈区数据由编译器自动释放（函数结束就释放 掉了)

3.4 堆区

自行分配和释放，程序结束后由操作系统回收

4 函数

  • 默认参数
  • 占位参数
  • 函数重载

4.1 默认参数

  1. 默认参数的优先级低于实参的优先级（默认参数位置传入实参时，默认参数被覆盖）
  2. 从默认参数位置起，其后的所有参数都必须为默认参数。
  3. 函数的声明与定义分开时，其中只能有一个拥有默认参数（建议在声明中写）

4.2 占位参数

  1. 占位参数即只指定函数形参的数据类型，不指定形参名
  2. 占位参数也可以使用默认参数

4.3 函数重载

  1. 函数重载条件
    • 函数名相同
    • 函数在同一个作用域下
    • 函数形参列表的形参数据类型不同 / 个数不同 / 顺序不同
  2. 引用作为函数重载条件
  3. 函数重载尽量不要使用默认参数

4.3.1 重载条件

Note 函数的返回值不能作为函数重载的条件。当只进行函数简单调用时， 因为参数相同，同名的函数都可以被调用，所以不允许。

4.3.2 引用作为重载条件

Note 引用可以作为函数重载的条件，在使用时要注意 const；但不代表 const 修饰的形参可以做为函数重载的条件，即，形参分别为 const a 和 a 的同名且 作用域相同的两个函数，在使用时会被判定函数重定义

4.3.3 尽量不要使用默认参数
